# Experiences

**Tensor indexing (strides)** — `TensorN` now keeps a row-major stride per axis (`m_strides`), recomputed whenever the shape changes. Linear index is `sum_i index[i] * stride[i]` instead of rebuilding tail products with `std::accumulate` on every access. Helpers like slice `assign` reuse cached strides from the source tensor; `linearToRowMajorIndices` takes strides directly. Profiling showed this removed most of the cost that looked like “slow `loopIndices`”: the visitor was fine; repeated \(O(\text{rank}^2)\) index work was not.

**Batch normalization convergence** — Adding `BatchNormLayer` (Conv → BN → ReLU) to the CIFAR-10 momentum SGD demo produced a dramatic improvement in convergence speed. Loss dropped from ~2.0 at epoch 1 without BN to ~0.6 → ~0.4 → ~0.1 across the first three epochs with BN. The mechanism: BN normalizes each channel's activation distribution (zero mean, unit variance) over the batch and spatial dimensions before the nonlinearity, eliminating internal covariate shift. This stabilizes gradients across layers and allows each layer to learn against a consistent input distribution rather than chasing a moving target. In practice the optimizer integrates naturally — `BatchNormLayer` exposes `getWeights()`/`getBias()` as aliases for γ/β, so no optimizer changes were needed. The `momentum` parameter (EMA factor for running mean/variance) controls how quickly the stored inference statistics track the training distribution; `eps` avoids divide-by-zero in low-variance channels.
